# Security Policy

## Supported Versions

Security fixes target the current `stable` branch and may also land on `main`
before promotion.

## Reporting A Vulnerability

Please do not open a public issue for suspected vulnerabilities.

Use GitHub private vulnerability reporting for this repository if it is
available. If private reporting is not available, contact a project maintainer
privately and include:

- A short description of the issue.
- Steps to reproduce.
- The affected branch, commit, or release.
- Any relevant logs or screenshots with secrets removed.

We will acknowledge valid reports as quickly as practical and coordinate a fix
before public disclosure.
